# Shared Vocabulary — MyClaude Creator Engine

> Canonical definitions used consistently across all agents, skills, and references.
> When in doubt about terminology, defer to this file.

---

## Core Concepts

| Term | Definition | Used By |
|------|-----------|---------|
| **Creator** | Any person using the Engine to create products for MyClaude | All agents, all skills |
| **Product** | Any artifact publishable on MyClaude marketplace. One of 9 categories. | All agents, all skills |
| **MCS** | MyClaude Creator Spec — 3-tier quality system (MCS-1: Publishable, MCS-2: Quality, MCS-3: State-of-the-Art). Scoring: (DNA×0.50)+(Structural×0.30)+(Integrity×0.20). | /validate |
| **Scaffold** | Auto-generated project structure for a product type, with WHY comments | /create |
| **WHY Comment** | `<!-- WHY: D{N} ({Name}) — {explanation} -->` comment explaining a DNA pattern section. Stripped during /package. | /create, /package |
| **Workspace** | `workspace/` directory where products are actively built. Gitignored. One dir per product. | All skills |
| **Package** | Distribution-ready directory (.publish/) with product files + vault.yaml + plugin.json + README + LICENSE | /package |
| **vault.yaml** | Machine-readable manifest consumed by CLI and marketplace. Source of truth for product identity. | /package, /publish |
| **plugin.json** | Anthropic Plugin Marketplace manifest. Generated alongside vault.yaml for dual distribution. | /package |
| **Activation Protocol** | Defined sequence of context-loading steps executed before responding (DNA Pattern D1) | All products |
| **Anti-Commodity Gate** | 3-question coaching check at MCS-2+ verifying genuine creator expertise. Advisory only, never blocking. | /validate Stage 7 |
| **Engine** | This system — MyClaude Studio Engine | CLAUDE.md |
| **creator.yaml** | Creator profile: expertise, goals, preferences. Generated by /onboard. | /onboard, all skills |
| **.meta.yaml** | Per-product state file. Tracks: product.type, state.phase, mcs_target, validation scores, history. | /create, /fill, /validate, /package |
| **Structural DNA** | 18 patterns in 3 tiers that define state-of-the-art Claude Code products. See structural-dna.md. | /validate, /create |
| **product-dna/** | Per-type DNA requirement files (9 YAML). Define which patterns are required vs optional per product type. | /validate, /create |

## Quality Tiers

| Tier | Name | Meaning | Gate |
|------|------|---------|------|
| MCS-1 | Publishable | Minimum viable quality — structure, metadata, no broken refs | Automated |
| MCS-2 | Quality | Craft demonstrated — exemplars, anti-patterns, tested, no placeholders | Semi-automated + Anti-Commodity Gate |
| MCS-3 | State-of-the-Art | Expert validated — deep knowledge, composable, stress-tested, differentiated | Quality Sentinel agent (PRO) |

## Skill Handoff Protocol

When one skill's output feeds into another (e.g., `/validate` detecting low differentiation
and suggesting `/differentiate`), the handoff context SHOULD include:

```yaml
handoff:
  from: "validate"              # skill name that identified the need
  to: "differentiate"           # skill name being recommended
  product_path: "workspace/{slug}/"
  creator_profile: "{loaded from creator.yaml}"
  findings:
    - "Anti-Commodity Gate: uniqueness score 35/100"
    - "No differentiation statement found"
  recommendation: "Run /differentiate to strengthen market positioning"
  context: "Product failed MCS-2 differentiation check"
```

Note: Skills cannot invoke each other directly. The handoff is informational —
Claude presents the recommendation and the creator decides whether to proceed.

## Validation States

| State | Meaning | Set By |
|-------|---------|--------|
| `scaffolded` | Structure created, content not yet filled | scaffolder |
| `in-progress` | Creator is actively building content | creator (manual) |
| `validated` | Last validation run passed at target MCS level | validator |
| `packaged` | Product bundled into distribution ZIP | publisher |
| `published` | Product submitted to marketplace | publisher |
